# Lego direction and head-color model

This folder contains three steps:

1. `label_dataset.py` uses CLIP to create automatic pseudo-labels for all 494 images. It reads `index.csv` and `test.csv`, and includes additional JPGs from character folders by their unambiguous folder-to-character mapping. Manual review is optional.
2. `train_model.py` fine-tunes a two-head MobileNetV3 model and writes SafeTensors artifacts.
3. `d3.py` predicts one image and saves an annotated copy in `output/`.

Use the project's virtual environment from the repository root:

```powershell
& ".\.venv\Scripts\python.exe" -m pip install -r "Day 3\requirements.txt"
& ".\.venv\Scripts\python.exe" "Day 3\label_dataset.py"
& ".\.venv\Scripts\python.exe" "Day 3\train_model.py"
& ".\.venv\Scripts\python.exe" "Day 3\d3.py"
```

To continue training on top of the best existing checkpoint, run:

```powershell
& ".\.venv\Scripts\python.exe" "Day 3\train_model.py" --resume --epochs 10
```

Resume mode unfreezes the complete network and uses one tenth of the normal learning rate. It restores model weights and training history, but starts a fresh optimizer so no unsafe pickle-based optimizer artifact is needed.

## Live human-to-Lego matcher

Run the separate webcam matcher with:

```powershell
& ".\.venv\Scripts\python.exe" "Day 3\live_matcher.py"
```

The first launch uses CLIP to build one visual prototype for each of the 38 Lego identities. It removes exact duplicate images and caches the result in `artifacts/lego_match_index.safetensors`. Later launches reuse that index.

To build or rebuild the index without opening the camera, use:

```powershell
& ".\.venv\Scripts\python.exe" "Day 3\live_matcher.py" --rebuild-index --build-index-only
```

The camera stays live while matching happens in a background thread. The side panel shows the top three visual matches and their reference images. Press `Q` to quit, `S` to save the current result, or `R` to rebuild the Lego index. Similarity is a playful visual comparison, not facial recognition or identity confidence.

The default dataset location is:

```text
%USERPROFILE%\.cache\kagglehub\datasets\ihelon\lego-minifigures-classification\versions\30
```

Pass `--dataset-root` to the labeling and training commands if it moves. Label generation resumes from `labels.csv`, and review progress is saved after every image.

The training command intentionally stops if labeling did not finish, if character IDs leak between splits, or if a required output class is missing from the training split. Because the labels are generated by CLIP, reported test scores measure agreement with CLIP rather than true real-world accuracy.

Manual review is not required. If you ever choose to run `label_dataset.py --review required`, press Enter or `1` when both labels are correct. Choose `2` for an incorrect direction, `3` for an incorrect color, or `4` when both are incorrect.

## Sports-ball odd-one-out game

`sportballs.py` downloads (or reuses) the 15-class sports-ball dataset and opens a
four-picture desktop game. Three pictures show the same kind of sports object and
one is different. The model secretly makes its own choice before you click. After
your answer, the grid labels the correct picture, your pick, and the model's pick;
the scoreboard tracks both players. The model then learns from the four real folder
labels. Your click is never used as a training label.

Use the **Mode** button above the pictures to switch between Normal and Hard. Hard
mode keeps four pictures but chooses the odd image from a deliberately similar
class: for example rugby versus American football, baseball versus cricket, or
tennis versus table tennis. The selected mode applies to the next round.

Install the requirements and launch the game from the repository root:

```powershell
& ".\.venv\Scripts\python.exe" -m pip install -r "Day 3\requirements.txt"
& ".\.venv\Scripts\python.exe" "Day 3\sportballs.py"
```

The first launch performs a short three-epoch warm-up. Later launches resume from
`artifacts/sportsball_game/`. Progress is saved after every completed round.

Useful options:

```powershell
# Use an already-downloaded copy instead of asking kagglehub for its path
& ".\.venv\Scripts\python.exe" "Day 3\sportballs.py" --dataset-root "C:\path\to\dataset"

# Erase only the sports-ball checkpoint and begin a fresh game
& ".\.venv\Scripts\python.exe" "Day 3\sportballs.py" --reset

# Make round selection and initial sampling reproducible with another seed
& ".\.venv\Scripts\python.exe" "Day 3\sportballs.py" --seed 123
```

If a saved checkpoint is incomplete, corrupt, or belongs to a different class list,
the program reports the problem and safely starts a fresh warm-up. Closing the
window during background training may discard only the unfinished round update.
